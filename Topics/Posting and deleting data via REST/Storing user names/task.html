This problem is fully supported by version 2021.9 of the EduTools plugin and newer. To avoid possible errors, please make sure that the plugin is updated. Alternatively, you can solve it using the web interface.

Implement a program that stores user names. Your program should contain two endpoints:

  • POST /users: takes the query parameter name that represents the user name and stores the user name;
  • GET /users: returns an array of user names in the order they were added. If there are no users returns an empty array.

In both cases, the server should respond with 200 (OK) status code.

Example 1: Response for a GET /users request:

[]

Example 2:

POST /users/?name=Alice
POST /users/?name=Kate
POST /users/?name=Someone

Example 3: Response for a GET /users request:

[
   "Alice",
   "Kate",
   "Someone"
]
