This problem is fully supported by version 2021.9 of the EduTools plugin and newer. To avoid possible errors, please make sure that the plugin is updated. Alternatively, you can solve it using the web interface.

Currently, the program contains the following endpoint:

  • GET /items/{id} — returns an item with a specified id and responds with 200 (OK). If an item is not present, it returns the message "Item wasn't found" and also responds with 200 (OK).

Your task is to modify the program — it should respond with 404 Not Found status code when an item with a specified id is not found.

Do not change the content of the Map.

Example 1:

Request: GET /items/1

Status code: 404 Not Found

Example 2:

Request: GET /items/999

Status code: 404 Not Found
